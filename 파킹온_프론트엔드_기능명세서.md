## 1. 로그인 페이지

### 페이지 정보
- **파일**: `login.html`
- **URL**: `/login`
- **설명**: 사용자 인증 페이지

### UI 구성요소

#### 1.1 로고 영역
- 파킹온 로고 이미지 표시
- 위치: 상단 중앙

#### 1.2 로그인 폼
**입력 필드:**
- 아이디 입력란
  - 타입: text
  - 필수 입력
  - placeholder: "아이디 입력"
  - 유효성 검증: 공백 체크

- 비밀번호 입력란
  - 타입: password
  - 필수 입력
  - placeholder: "비밀번호"
  - 유효성 검증: 공백 체크

**버튼:**
- 로그인 버튼
  - 타입: submit
  - 액션: POST `/login`

- 비밀번호 변경 링크
  - 클릭 시: 알림창 표시 ("관리자에게 문의하세요.")

### 프론트엔드 구현 요구사항
1. 입력 필드 포커스 시 테두리 색상 변경
2. 유효성 검증 실패 시 에러 메시지 표시
3. Enter 키로 로그인 가능
4. 로그인 실패 시 에러 메시지 표시

## 2. 대시보드 (멀티스크린)

### 페이지 정보
- **파일**: `dashboard.html`
- **URL**: `/dashboard`
- **설명**: 전체 시스템 현황을 한눈에 볼 수 있는 통계 대시보드

### UI 구성요소

#### 2.1 차트 영역

**공통 날짜 선택기**
- 드롭다운 형태
- 기본값: 오늘 날짜
- 날짜 변경 시 모든 차트 데이터 갱신

**상담 VOC 지역 분포 차트**
- 타입: 도넛/파이 차트
- 데이터: 수도권, 강원권, 충청권, 남부권, 제주권
- 라이브러리: Chart.js

**상담 VOC 인입 분포 차트**
- 타입: 도넛/파이 차트
- 데이터: 출근중, 휴근중, 담당배제, 내다주, 담당중, 이행중
- 라이브러리: Chart.js

#### 2.2 단지 현황 카드
- 총 단지 수
- 정상 운영 단지 (초록색 강조)
- 장애 단지 (빨간색 강조)

#### 2.3 VOC 고객구분 통계
**표시 항목:**
- 전체, 일반, 입주민, 주기/방문, 방문, 출역, 화이트
- 각 항목별 건수 표시

#### 2.4 VOC 발생 목적 통계
**표시 항목:**
- 전체, 방문, 미인식, 관동, 기타
- 각 항목별 건수 표시

#### 2.5 입출차 현황 테이블
**기능:**
- 월 선택 드롭다운 (YYYY-MM 형식)
- 1~31일까지 가로 스크롤 테이블
- 입차 섹션: 입주민, 방문, 블랙, 화이트, 주기/방문
- 출차 섹션: 출차 합계
- 각 셀에 해당 날짜의 입출차 건수 표시

#### 2.6 신규/폐쇄 현황 차트
- 타입: 바 차트
- X축: 월 (1~12월)
- Y축: 단지 수
- 데이터: 신규 단지, 폐쇄 단지
- 연도 선택 드롭다운
- 라이브러리: Chart.js

#### 2.7 공지사항 팝업
- 페이지 로드 시 자동 표시 (조건: 활성화된 공지사항 존재)
- 공지사항이 여러 개인 경우 순차 표시
- 다음/이전 버튼으로 네비게이션
- 닫기 버튼

### 프론트엔드 구현 요구사항
1. Chart.js를 사용한 차트 렌더링
2. 날짜/연도/월 선택 시 데이터 동적 갱신 (Ajax)
3. 입출차 테이블 가로 스크롤 구현
4. 반응형 레이아웃 (카드형 그리드)
5. 공지사항 팝업 자동 표시 및 순차 네비게이션

## 3. 대시보드 (근무자용)

### 페이지 정보
- **파일**: `dashboard-worker.html`
- **URL**: `/dashboard/worker`
- **설명**: 상담원이 실시간으로 입차 차량을 처리하는 페이지

### UI 구성요소

#### 3.1 좌측: 아파트 목록 사이드바
- 검색 입력란 + 조회 버튼
- 아파트 목록 테이블
  - 체크박스 (전체 선택 / 개별 선택)
  - 아파트 코드
  - 아파트명
  - 행 클릭 시 선택 상태 변경
- 페이지네이션

#### 3.2 중앙: 메인 패널

**상단 상태 표시줄**
- 차단기 상태 (정상/점검/오류)
- 시스템 상태
- CPU 사용률
- 메모리 사용률
- 통신 상태
- 저장소 사용량

**CCTV 영역**
- 실시간 CCTV 스트리밍
- 크기: 320×180px
- 플레이스홀더 표시

**입차 이미지 영역**
- 차량 입차 시 촬영된 이미지
- 크기: 320×180px
- 플레이스홀더 표시

#### 3.3 우측: 컨트롤 패널

**차량번호 표시**
- 인식된 차량번호 크게 표시
- 예: "12가1234"

**이전 상담내역**
- 최근 상담 2건 표시
- 시간 + 상담 내용 요약

**상담 정보 입력 폼**
- 단지명 (읽기 전용)
- 일시 (읽기 전용, datetime-local)
- 차량번호 (수정 가능)
- 고객유형 (드롭다운: 방문고객, 입주민, 출역)
- 방문목적 (드롭다운: 방문, 배달, 택배)
- 동/호 (2개의 드롭다운)
- 상담내역 (textarea)

**차단기 제어**
- 라디오 버튼: 열림/닫힘
- 차단기 제어 버튼
  - 클릭 시: 선택된 상태로 차단기 제어 요청

#### 3.4 하단 상태 바
- 방문차단이력 상태
- VOC이력 상태
- 수동제어 상태
- 검지차 상태
- 상태별 칩 컴포넌트 (정상/경고/오류)

#### 3.5 PBX 바
- PBX 호 정보 표시
- 예: "PBX콜: 동아아파트"

### 프론트엔드 구현 요구사항
1. 실시간 데이터 갱신 (WebSocket 또는 폴링)
2. CCTV 스트리밍 (RTSP 또는 HLS)
3. 차량번호 자동 인식 시 폼 자동 채우기
4. 차단기 제어 API 호출
5. 상담내역 저장 시 유효성 검증
6. 상태 변화에 따른 색상 변경 (정상: 초록, 경고: 노랑, 오류: 빨강)

## 4. APT 관리

### 4.1 아파트 단지 관리

#### 페이지 정보
- **파일**: `apt/apt-manage.html`
- **URL**: `/apt/manage`
- **설명**: 아파트 단지 정보 등록 및 수정

#### UI 구성요소

**좌측: 아파트 목록 사이드바**
- 검색 + 조회 버튼
- 아파트 목록 테이블 (체크박스, 코드, 명칭)
- 페이지네이션

**우측: 아파트 정보 폼**

**기본 정보 섹션**
- 아파트명 + 추가 버튼
- 아파트 코드 + 중복확인 버튼
- 권역 (드롭다운: 수도권, 강원권, 충청권, 남부권, 제주권)
- 관리소명
- 담당자명
- 연락처
- 영업담당자
- 영업담당자 연락처
- 우편번호 + 찾기 버튼 (다음 주소 API 연동)
- 주소
- 메모 (textarea)

**운영 설정 섹션**
- 업무 시작 시간 (time input) + 기본값 버튼
- 업무 종료 시간 (time input)
- 차단기 (드롭다운: 정상 운영, 점검중, 사용안함)
- 블랙 리스트 (드롭다운: 입차 제한, 경고만, 사용안함)
- 화이트 리스트 (드롭다운: 자동 입차, 확인 후 입차, 사용안함)
- 주기적 방문차량
  - N일 이내 (드롭다운: 10, 20, 30)
  - M회 이상 입차 (드롭다운: 1, 2, 3)
  - 처리 방식 (드롭다운: 자동 입차, 확인 후 입차, 사용안함)

**액션 버튼**
- 초기화 버튼
- 저장 버튼

#### 프론트엔드 구현 요구사항
1. 아파트 코드 중복확인 Ajax 검증
2. 다음 주소 API 연동
3. 기본값 버튼 클릭 시 시스템 설정값으로 자동 입력
4. 폼 유효성 검증 (필수 입력 체크)
5. 저장 성공/실패 시 알림 메시지
